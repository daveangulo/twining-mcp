# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Twining, please report it through [GitHub Security Advisories](https://github.com/daveangulo/twining-mcp/security/advisories/new).

**Do not** open a public issue for security vulnerabilities.

## Scope

The following are in scope:

- The Twining MCP server (`src/`)
- The Claude Code plugin (`plugin/`)
- The web dashboard (`src/dashboard/`)
- File storage and locking mechanisms
- Any data exposure through the MCP protocol

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Resolution target**: Within 14 days for confirmed vulnerabilities
- **Disclosure**: Coordinated disclosure after a fix is available

## Supported Versions

Only the latest published version on npm receives security updates.

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | No        |
