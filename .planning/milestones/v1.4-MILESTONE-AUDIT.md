### Integration Issues (Non-Blocking)

1. **eval:baseline ordering fragility** — `regression-baseline.ts` uses `readJson()` (throws) for `latest.json` but `readOptionalJson()` for holdout/transcript. Running `eval:baseline` before `eval:synthetic` crashes. Fix: add `readOptionalJson` fallback or pre-hook.

2. **Holdout console leakage** — `vitest.config.eval.ts` pattern `test/eval/**/*.eval.ts` matches `holdout-runner.eval.ts`, so `eval:synthetic` runs holdout tests too. Data isolation is correct (separate JSON files) but terminal output shows holdout scores. Fix: rename holdout runner or add exclude pattern.

3. **Token script not in npm** — `scripts/measure-plugin-tokens.sh` works but has no npm script. The regression baseline embeds token data inline, so the baseline is complete. Fix: add `npm run eval:tokens` script.
