   - Reads test/eval/transcript-manifest.json to discover fixtures
   - For each fixture in manifest:
     - Reads the JSONL file from the path relative to test/eval/
     - Calls parseTranscript() to get ParseResult with segments
     - For each segment, runs all 7 scorers from allScorers
     - Uses the manifest's threshold (default 0.6) instead of DEFAULT_THRESHOLD (0.8) for pass/fail
     - Generates vitest test cases: `describe(fixtureName) > describe(segment-N) > it(scorerName)`
   - In afterAll: writes accumulated results to test/eval/results/transcript-latest.json in the same JSON format as eval-runner.eval.ts (timestamp, summary, scenarios array), but with fixture/segment naming instead of scenario naming
   - Load BehaviorSpec from plugin/BEHAVIORS.md in beforeAll (same as eval-runner.eval.ts)
   - For each fixture x segment x scorer test, the test should:
     - Run the scorer
     - Accumulate the result
     - If the fixture has expectedQuality "poor", do NOT assert pass (poor fixtures are expected to fail some scorers)
     - If the fixture has expectedQuality "good", assert result.score >= threshold (0.6)
     - Always log the result regardless of assertion

   Structure the test output for clarity:
   ```
   describe("transcript eval suite")
     describe("session-good-workflow.jsonl")
       describe("segment 0 (12 calls)")
         it("sequencing") ...
         it("scope_quality") ...
         ...
     describe("session-poor-workflow.jsonl")
       describe("segment 0 (5 calls)")
         ...
   ```
