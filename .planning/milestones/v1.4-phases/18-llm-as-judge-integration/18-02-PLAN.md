**Create `test/eval/scorers/rationale-judge.ts`:**

This scorer evaluates the quality of `rationale` and `context` arguments in `twining_decide` tool calls.

1. Import `Scorer`, `ScorerResult`, `CheckResult` from `../scorer-types.js`, `ScorerInput` from `../scenario-schema.js`, `BehaviorSpec` from `../types.js`, and `createJudgeClient`, `callJudge`, `consensusScore`, `buildRubricPrompt` from `../judge.js`.

2. Implement `rationaleJudgeScorer: Scorer`:
   - `name: "rationale-judge"`
   - `async score(input, spec)`:
     a. Find the "rationale-quality" criterion from `spec.qualityCriteria` (by name match). If not found, return vacuous pass with `type: 'llm'`.
     b. Filter `input.calls` to only `twining_decide` calls. If none, return vacuous pass.
     c. Call `createJudgeClient()`. If null (no API key), return vacuous pass with a check message noting "LLM judge skipped: no API key".
     d. For each decide call, build a context string containing:
        - The tool name and call index
        - The `rationale` argument value (or "not provided")
        - The `context` argument value (or "not provided")
        - The `summary` argument value for additional context
        - Scenario metadata if available from `input.metadata`
     e. Use `buildRubricPrompt(criterion, contextString)` to construct the prompt.
     f. Call `consensusScore(() => callJudge(client, prompt))` to get the consensus result.
     g. Convert each consensus result to a `CheckResult`:
        - `ruleId`: `"JUDGE-rationale-{call.index}"`
        - `level`: `"SHOULD"` (semantic quality is advisory)
        - `passed`: consensus score >= 0.5 (acceptable or good)
        - `message`: Include the consensus level, agreement ratio, and first trial's reasoning
     h. Wrap the whole scoring in try/catch. On error, log a warning (`console.warn`) and return vacuous pass with a check noting the error.
     i. Aggregate checks, return `ScorerResult` with `type: 'llm'`, `passed` at `DEFAULT_THRESHOLD`.

**Create `test/eval/scorers/scope-judge.ts`:**

This scorer evaluates whether scope arguments are contextually appropriate for the task.

1. Same imports pattern as rationale-judge.

2. Implement `scopeJudgeScorer: Scorer`:
   - `name: "scope-judge"`
   - `async score(input, spec)`:
     a. Find the "scope-precision" criterion from `spec.qualityCriteria`. If not found, return vacuous pass with `type: 'llm'`.
     b. Filter `input.calls` to those with a string `scope` argument. If none, return vacuous pass.
     c. Call `createJudgeClient()`. If null, return vacuous pass with skip message.
     d. For each scoped call, build a context string containing:
        - The tool name and call index
        - The `scope` argument value
        - Other arguments for context (e.g., `summary`, `category` for decide; `summary` for post; `query`/`scope` for assemble)
        - Scenario metadata if available
     e. Use `buildRubricPrompt(criterion, contextString)` to construct the prompt.
     f. Call `consensusScore(() => callJudge(client, prompt))`.
     g. Convert to `CheckResult`:
        - `ruleId`: `"JUDGE-scope-{call.index}"`
        - `level`: `"SHOULD"`
        - `passed`: consensus score >= 0.5
        - `message`: Include consensus level, agreement ratio, reasoning
     h. Wrap in try/catch with warning and vacuous pass on error.
     i. Aggregate checks, return `ScorerResult` with `type: 'llm'`.

**Both scorers must:**
- Return `type: 'llm'` on every ScorerResult
- Gracefully handle missing API key (vacuous pass)
- Gracefully handle API errors (warn + vacuous pass)
- Use `DEFAULT_THRESHOLD` for pass/fail
- Use `aggregateChecks` for score computation
- Process multiple relevant calls independently (one judge call per tool call, not batched)
