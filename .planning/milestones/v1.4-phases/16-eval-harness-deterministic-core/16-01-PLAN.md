  <files>test/eval/scenario-schema.ts, test/eval/scorer-types.ts</files>
  <behavior>
    - ScenarioSchema validates YAML with required fields: name, description, category (enum of orient/decide/verify/coordinate/handoff), tags array, tool_calls array (each with tool string starting with "twining_" and arguments record), expected_scores record of boolean
    - ScenarioSchema rejects invalid category values
    - ScenarioSchema rejects empty tool_calls arrays
    - NormalizedToolCall has tool, arguments, and index fields
    - ScorerInput has calls (NormalizedToolCall[]) and optional metadata
    - Scorer interface has name string and score(input: ScorerInput, spec: BehaviorSpec) returning ScorerResult
    - ScorerResult has scorer name, numeric 0-1 score, passed boolean, and checks array of CheckResult
    - CheckResult has ruleId, level, passed, and message
    - aggregateChecks function: MUST fail=0, SHOULD fail=0.5, MUST_NOT fail=0, pass=1, mean of all
    - aggregateChecks with empty checks returns 1.0
  </behavior>
  <action>
    Create two type definition files that form the contracts for the entire eval system.

    **test/eval/scenario-schema.ts:**
    - Import z from "zod"
    - Define ScenarioSchema as a Zod object: name (string min 1), description (string min 1), category (z.enum of the 5 workflow categories), tags (array of string, default []), tool_calls (array of objects with tool string starting with "twining_" and arguments record of unknown defaulting to {}, min 1 item), expected_scores (record of boolean, default {})
    - Export Scenario type inferred from ScenarioSchema
    - Define and export NormalizedToolCall interface: tool (string), arguments (Record<string, unknown>), index (number)
    - Define and export ScorerInput interface: calls (NormalizedToolCall[]), metadata? (Record<string, unknown>)
    - Export a normalizeScenario function that converts a Scenario's tool_calls into ScorerInput (adds index to each call)

    **test/eval/scorer-types.ts:**
    - Import BehaviorSpec from "./types"
    - Import ScorerInput from "./scenario-schema"

    - Define and export ScorerResult interface: scorer (string), score (number), passed (boolean), checks (CheckResult[])
    - Define and export Scorer interface: name (string), score(input: ScorerInput, spec: BehaviorSpec): ScorerResult
    - Export aggregateChecks function implementing weighted severity: empty checks = 1.0, passed = 1.0, MUST/MUST_NOT fail = 0, SHOULD fail = 0.5, return mean rounded to 4 decimal places
