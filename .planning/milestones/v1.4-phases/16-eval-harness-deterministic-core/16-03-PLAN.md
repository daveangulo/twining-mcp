    **test/eval/eval-runner.eval.ts:**
    - Import describe, it, expect, beforeAll, afterAll from "vitest"
    - Import loadScenarios from "./scenario-loader"
    - Import normalizeScenario from "./scenario-schema"
    - Import allScorers from "./scorers"
    - Import parseBehaviors from "./behaviors-parser"
    - Import fs from "node:fs" and path from "node:path"
    - Import BehaviorSpec from "./types"

    In the file's top-level scope:
    - Call loadScenarios() to get all scenarios (this runs at collection time)

    In describe("eval suite"):
    - beforeAll: parse BEHAVIORS.md into spec using parseBehaviors(fs.readFileSync("plugin/BEHAVIORS.md", "utf-8"))
    - Define a results accumulator: Map<string, { scenario: string, results: ScorerResult[] }>

    - For each scenario, create a describe(scenario.name) block:
      - For each scorer in allScorers, create it(`${scorer.name}`) test:
        - Call scorer.score(normalizeScenario(scenario), spec)
        - Store result in accumulator
        - If scenario.expected_scores has this scorer name: expect(result.passed).toBe(expected value)
        - If scorer not in expected_scores: skip assertion (test passes, result still collected)

    - afterAll: Write accumulated results to test/eval/results/latest.json as JSON with:
      - timestamp: ISO string
      - summary: { total_scenarios, total_checks, passed_checks, overall_score (mean of all scorer scores) }
      - scenarios: array of { name, category, scores: { [scorer]: { score, passed, checks_passed, checks_total } } }
    - Use path.resolve(import.meta.dirname, "results/latest.json") for output path

    **test/eval/eval-reporter.ts:**
    - Export default class EvalReporter (custom vitest reporter)
    - Extend from vitest/reporters BaseReporter
    - Override onTestRunEnd(testModules): after all tests complete, read test/eval/results/latest.json (if it exists), and print a summary table to console:
      - Header: "Eval Summary" with run timestamp
      - Per-category row: scorer name | scenarios tested | pass rate | avg score
      - Overall row: total scenarios | total pass rate | overall avg score
      - Worst performers section: list the 3 scenarios with lowest average scorer scores
    - Use simple console.log formatting (no external table library)
    - Handle gracefully if latest.json doesn't exist yet (first-run scenario)

    **test/eval/results/.gitkeep:**
    - Empty file to ensure the results directory is tracked by git
    - Add test/eval/results/latest.json to .gitignore (generated file, not tracked)

    **vitest.config.eval.ts** (update from Plan 01):
    - Add the custom reporter: reporters: ["default", "./test/eval/eval-reporter.ts"]
    - Keep include: ["test/eval/**/*.eval.ts"] and testTimeout: 30000

    Also append to .gitignore:
    - `test/eval/results/latest.json`
